# skill-translate-game

Перевожу файлы `translations/dialogues.{path_id}.yaml`, `speakers.yaml`, `settings_keys.yaml`.

## Когда использовать

Загружай этот скил когда нужно перевести строки игры Third Crisis Neon Nights с английского на русский. Это **эротическая визуальная новелла / RPG** на Unity — диалоговые реплики с портретами персонажей, звукоподражания (SFX) внутри текста, наррация и UI-элементы. Скил задаёт тон (жёсткая эротика, не любовный роман), правила согласования рода, адаптации стонов и сексуальной лексики.

## Формат данных

```
# Dialogues (path_id=73203): text, translation, speaker, rich_text, rich_translation

- text: "Yesss...!~"
  translation: "Да-а-а...!~"
  speaker: "Zoey"
  rich_text: "<color=#B867FF><font=\"Roboto-Condensed_DialogueUI\" material=\"Roboto-Condensed_DialogueUI_Perversion\">Yesss...!~</font></color>"
  rich_translation: "<color=#B867FF><font=\"Roboto-Condensed_DialogueUI\" material=\"Roboto-Condensed_DialogueUI_Perversion\">Да-а-а...!~</font></color>"
- text: "Fhaaa..!!"
  translation: "Ахха..!!"
  speaker: "Zoey"
  rich_text: "Fhaaa..!!"
  rich_translation: "Ахха..!!"
```

- `text` — оригинал (не трогать)
- `translation` — перевод (заполнить)
- `speaker` — говорящий (не трогать, справочник `speakers.yaml`)
- `rich_text` — оригинал с Unity-тегами (не трогать)
- `rich_translation` — перевод с Unity-тегами. **Перенеси структуру тегов из `rich_text` без изменений**, заменив только английский текст внутри на русский перевод. Даже если теги оборачивают 100% текста — копируй их целиком, не вырезай. Если `rich_text` без тегов — просто скопируй `translation`. Пример:

  ```
  rich_text: "<color=#B867FF><font=...>Yesss...!~</font></color>"
  rich_translation: "<color=#B867FF><font=...>Да-а-а...!~</font></color>"
  # НЕПРАВИЛЬНО: rich_translation: "Да-а-а...!~" (теги потеряны — в игре не будет свечения)
  ```

**Важно: кириллица работает внутри `<font material="...Perversion">`.**  
Ранее материал Perversion (GLOW_ON + UNDERLAY_ON) применялся только к глифам,
присутствующим в оригинальном атласе. Теперь кириллические глифы добавляются
в тот же шрифт через `TryAddCharacters()`, поэтому `<font material>` теги
работают одинаково для латиницы и кириллицы. При переводе **сохраняй все
оригинальные теги** из `rich_text` без изменений.

**Важно:** рантайм использует fallback: `rich_translation` → `translation` → `rich_text` → `text`.
Заполнив `rich_translation`, ты получишь цветной/выделенный перевод в игре.

### speakers.yaml

```
# Speakers: text, translation, gender, notes
- text: "Zoey"
  translation: "Зои"
  gender: "female"
  notes: ""
```

- `text` — оригинал (не трогать)
- `translation` — имя по-русски
- `gender` — пол (female/male/пусто)
- `notes` — примечания

### settings_keys.yaml

```
# Settings keys: text, translation
- text: "Fullscreen"
  translation: "Полный экран"
```

## Особенности формата

### Типы текста в игре

| Тип                                | Описание                                                | Ограничения                       |
| ---------------------------------- | ------------------------------------------------------- | --------------------------------- |
| **Dialogue** — диалоговая реплика  | Текст в диалоговом окне (имя + текст), 1-3 строки       | **Не более 2× от оригинала**      |
| **Narration** — наррация / подпись | Текст без говорящего (пустой speaker), описание событий | Обычно длиннее диалога            |
| **Choice** — вариант выбора        | Кнопка/строка в меню выбора                             | **Максимум 25-30 символов**       |
| **UI** — текст интерфейса          | Кнопки, заголовки настроек                              | **Максимум 30-35 символов**       |
| **SFX** — звукоподражание в тексте | Слова вроде `*slurp*`, `*moan*` внутри реплик           | Те же 2×, капс для громких звуков |

### Ключевое: длина строки

**Золотое правило:** перевод сохраняет **полный смысл** оригинала. Слова и интонации не выбрасываются. Допускается увеличение длины относительно оригинала, но **не более чем в 2 раза**. Если оригинал 20 символов — перевод максимум 40.

- Оригинал `"You like that, don't you?"` (26 символов) → `"Тебе нравится, да?"` (19 символов) — **смысл сохранён**
- Оригинал `"Oh god yes, fuck me harder!"` (29 символов) → `"О да, трахни меня сильнее!"` (26 символов) — **смысл сохранён**
- Оригинал `"Please don't stop, I'm so close!"` (33 символа) → `"Пожалуйста не останавливайся, я близко!"` (41 символ) — **в пределах 2×**

**Если перевод выходит за 2×** — сокращай: более короткий синоним (`трахни` → `имей`), убери вводные слова (`you know`, `well`), но не выбрасывай смысловые слова (глаголы, существительные, ключевые прилагательные).

Причина: текст отображается в диалоговом окне фиксированной ширины. Длинный перевод может не влезть в окно, обрезаться или наложиться на UI.

### SFX — звукоподражания в тексте

В игре SFX встречается **внутри диалоговых реплик** как обычный текст, а не как графический элемент. Пример: `"Ahhh! *slurp* *slurp* Yesss!"` — это часть текста диалога. Поэтому SFX подчиняется тем же правилам длины (2×), что и обычный диалог, за исключением случаев, когда SFX явно вынесен в отдельную строку как реплика (тогда — как Shout).

| Английский SFX       | Русский аналог       | Примечание            |
| -------------------- | -------------------- | --------------------- |
| `SLURP` / `SLRRP`    | `ХЛЮП` / `ХЛЮ-П`     | Сосание, влажный звук |
| `SCHLUCK`            | `ГЛОТ`               | Глотание              |
| `POP`                | `ЧМОК`               | Вытащить изо рта      |
| `SQUELCH` / `SQUISH` | `ХЛЮП` / `ВХЛЮП`     | Проникновение, влага  |
| `WET SLAPPING`       | `ШЛЕП` / `ШЛЁП`      | Удары тел             |
| `SMACK`              | `ШМЯК`               | Удар/шлепок           |
| `THRUST`             | `ТОЛ-ЧОК`            | Толчок, движение      |
| `DRIP` / `DRIP DRIP` | `КАП` / `КАП-КАП`    | Капает                |
| `DROOL`              | `СЛЮ-НИ`             | Слюни, текут          |
| `PANT` / `HUFF`      | `ТЯ-ЖЕ-ЛО` / `ХА-АХ` | Тяжёлое дыхание       |
| `SLIDE`              | `ВХО-ДИТ`            | Плавное проникновение |
| `TWITCH`             | `ДЁРГ`               | Спазм/дёрганье        |
| `SHUDDER`            | `ДРОЖЬ`              | Дрожь                 |
| `CREAK`              | `СКРИП`              | Кровать скрипит       |
| `NGH` / `MNGH`       | `М-М` / `М-М-М`      | Стон, мычание         |
| `KISS` / `MWAH`      | `ЧМОК` / `М-УА`      | Поцелуй               |

**Правила SFX:**

- SFX в игре — текст, а не графика. Ограничение длины — общее (2×).
- Если SFX в оригинале в капслоке — сохраняй капслок: `SLURP` → `ХЛЮП`, а не `хлюп`.
- Сохраняй дефисы для разбивки на слоги: `THRUST` → `ТОЛ-ЧОК`, `DRIP DRIP` → `КАП-КАП`
- Не переводи SFX как обычные слова: `SLURP` ≠ «чавканье», а `ХЛЮП`. Это звук, не существительное.
- Если SFX обёрнут в \*звёздочки\* (`*slurp*`) — сохраняй звёздочки.

### Glitch / Zalgo / искажённый текст

В игре встречаются строки с **диакритическими комбинирующими символами** (zalgo), создающими эффект глитча, искажения или «порчи» текста:

```
Оригинал:   N̴o̴!̸ ̸G̶e̸t̷ ̷y̴o̶u̵r̴ ̷h̴a̷n̶d̶s̷ ̸o̴f̴f̷ ̴o̴f̷ ̸h̶i̶m̶.̴
```

Zalgo-символы — это не просто украшение, а **сюжетно значимый визуальный эффект**: одержимость, цифровая порча, трансформация, потеря контроля. Перевод должен сохранять этот эффект.

**Правила работы с zalgo:**

1. **Определи паттерн.** Оригинал использует определённый набор комбинирующих символов (COMBINING SHORT SOLIDUS OVERLAY `̸`, COMBINING LONG SOLIDUS OVERLAY `̶`, COMBINING SHORT STROKE OVERLAY `̵` и т.д.). Их комбинация — часть дизайна.

2. **Повтори паттерн на переводе.** Не просто переводи слова, а накладывай те же комбинирующие символы на каждую букву русского текста. Количество и расстановка символов должны совпадать с оригиналом.

3. **Как делать:**
   - Возьми оригинальную строку — **извлеки комбинирующие символы** из неё (U+0300–U+036F, U+0338 и т.п.)
   - Напиши перевод обычным текстом
   - На каждую букву перевода навесь **такой же набор** комбинирующих символов, как на соответствующей букве оригинала

4. **Если перевод короче оригинала** — последние символы оригинала тупо обрезать, они не понадобятся.

5. **Не добавляй zalgo**, если в оригинале его нет. Это сознательный дизайнерский приём, а не общее оформление.

6. **Исключение:** если переводчик не может корректно воспроизвести zalgo технически (не хватает инструментов/шрифта) — допустимо перевести обычным текстом, но это потеря качества. Zalgo-эффект рекомендуется восстанавливать при любом удобном случае.

Пример корректной работы:

```
Оригинал:  N̴o̴!̸ ̸G̶e̸t̷ ̷y̴o̶u̵r̴ ̷h̴a̷n̶d̶s̷ ̸o̴f̴f̷ ̴o̴f̷ ̸h̶i̶m̶.̴
            ̴ ̴ ̸+̸G̶ ̸ ̷ ̷ ̴ ̶ ̵ ̴ ̷ ̴ ̶ ̶ ̷ ̸ ̴ ̴ ̷ ̴ ̷ ̸ ̶ ̶ ̴
Перевод:   Н̸е̴т̴!̶ ̷У̸б̷е̶р̴и̵ ̶о̸т̷ ̴н̷е̵г̶о̴ ̷р̵у̴к̴и̷.̸
```

Или проще — сохранить **плотность и распределение** символов, даже если конкретные юникод-коды не совпадают один-в-один, лишь бы визуальный эффект был тем же.

### Адаптация текста под диалоговое окно

Текст отображается в диалоговом окне (нижняя часть экрана), обычно 1-3 строки за раз. Окно имеет фиксированную ширину, но может скроллиться. Поэтому:

- **Одна реплика = одна строка YAML.** Если оригинал слишком длинный — в исходном коде это уже разбито; не склеивай и не разрывай.
- **Эллипсис** — опускай слова, которые ясны из контекста и картинки. Арт и анимация уже показывают, что происходит, не нужно это проговаривать.
- **Восклицания** — сохраняй энергию.
- **Скобки** — ремарки в скобках `(whispering)` или `(sighs)`. Можно опустить, если эмоция понятна из контекста.
- **Курсив/звёздочки** `*текст*` — могут обозначать шёпот, мысль или ударение. Сохранять по возможности.

### Имена персонажей

Имя говорящего отображается в диалоговом окне отдельно от текста реплики. Поэтому внутри текста имя персонажа — это обращение к кому-то, а не подпись говорящего.

- `"Oh Zoey!"` → `"Зои!"` — обращение к Зои, не подпись.
- Не добавляй имя, если в оригинале его нет — оно уже есть в UI.
- Если в оригинале имя призывное и эмоциональное (`"Zoey... baby..."`) — переводи (`"Зои... детка..."`).
- **Склонение имён:** сверяйся с `speakers.yaml`. Имена на -и (Зои) не склоняются. Сара → Сары, Саре. Клавдия → Клавдии, Клавдию. Если сомневаешься — ставь в именительный падеж.

### Наррация (пустой speaker)

Строки с пустым `speaker` — это наррация: текст от лица игры, описывающий действия, обстановку или мысли персонажа.

- **Стиль:** литературный, 3-е лицо, настоящее время.
- **Нет гендера** — избегай глаголов прош. времени с родом. Используй настоящее время или инфинитив.
- **Пример:** `"The door opens slowly"` → `"Дверь медленно открывается"`, не `"открылась"`.
- **Наррация может быть длиннее диалога** — длина определяется описанием, а не репликой. Тем не менее, 2× желательно не превышать.

### Диалоговые выборы (choice)

В игре есть моменты, где игрок выбирает реплику из списка (меню выбора). Choice-текст имеет жёсткое ограничение — он отображается в кнопке/строке списка.

- **Максимум 25-30 символов.** Если оригинал длиннее — сокращай без потери смысла.
- **Стиль:** коротко, ёмко, часто императив (`"Согласиться"`, `"Отказаться"`).
- **Нет speaker** — выбор всегда от лица игрока.
- **В YAML choice находится в том же файле диалогов** и определяется контекстом (короткая строка, часто без speaker, часто как последняя строка перед блоком реакции).

## Ключевые правила

### 0. Атомарный перевод — один YAML-блок за раз (КРИТИЧНО)

**Единица перевода — один YAML-блок (одна реплика от `- text:` до следующего `- text:`).**
Не пытайся перевести весь файл за раз и даже несколько блоков подряд. Один блок = одна микросессия: прочитал → осмыслил → перевёл → проверил → перешёл к следующему.

Визуально граница блока:

```yaml
# НАЧАЛО БЛОКА (атом)

- text: "Fhaaa..!!"
  translation: "Ахха..!!"
  speaker: "Zoey"
  rich_text: "<color=#B867FF>Fhaaa..!!</color>"
  rich_translation: "<color=#B867FF>Ахха..!!</color>"
# КОНЕЦ БЛОКА — обработал, перешёл к следующему блоку
```

**Запрещается:**

- bulk-автозамена, массовый find-and-replace по всему файлу
- попытка перевести больше одного блока за раз
- автоматический/машинный перевод нескольких блоков одной командой
- применение одинаковой логики перевода к пачке блоков

**Почему это критично:**

- каждая реплика контекстуально уникальна (интонация, сцена, адресат)
- две одинаковых строки (`"Ahhh..."`) в разных сценах могут означать разное
- zalgo, rich_text-теги, спецсимволы требуют индивидуального внимания к каждому блоку
- при пачечном переводе качество резко падает — теряются нюансы, род, естественность

**Протокол работы (один цикл = ровно один блок):**

1. Прочитай ровно один YAML-блок (от `- text:` до пустой строки перед следующим `- text:`)
2. Прочитай оригинал (`text`), осмысли контекст (что за сцена, кто говорит, кому)
3. Если есть `speaker` — сверься с `speakers.yaml` (род, склонение имени)
4. Напиши `translation`
5. Заполни `rich_translation`: скопируй структуру тегов из `rich_text` без изменений, заменив только английский текст внутри на русский перевод. **Даже если теги оборачивают 100% текста — копируй их целиком, не вырезай**
6. Проверь длину (макс 2×), род глаголов, естественность звучания
7. Переходи к следующему блоку. Не оглядывайся назад, не забегай вперёд

**Режим ревью:**
Этот скил можно (и нужно) использовать для **повторного прогона** существующих переводов с целью улучшения качества:

1. Берёшь файл перевода — он уже заполнен, но может быть неидеален
2. Проходишь поатомарно: один блок → оригинал → существующий перевод → ок или не ок
3. Если перевод хороший — оставляешь, переходишь к следующему блоку
4. Если плохой — правишь прямо здесь (род, стон, длина, lost rich_text-теги)
5. Zalgo-блоки проверяешь особенно тщательно — восстановлены ли символы
6. rich_translation проверяешь: есть ли теги, не потеряны ли они

Режим ревью — тот же поатомарный подход, но с фокусом на «ок или не ок» вместо «заполнить пустоту».

### 1. Род глаголов (КРИТИЧНО)

Английский не различает род в прошедшем времени (`you saw`). Русский — различает.

**Адресат — Zoey (она):**

- `"Ты видел его?"` — НЕТ, мужской род
- `"Ты видела его?"` — ДА

**Адресат — мужчина:**

- `"Ты была права"` — НЕТ, женский род
- `"Ты был прав"` — ДА

**Неизвестный адресат — нейтрализация:**

- настоящее время: `"Ты видишь его?"`
- безличное: `"Тебе довелось увидеть?"`

| Приём           | Вместо     | Используй               |
| --------------- | ---------- | ----------------------- |
| Настоящее время | «Ты видел» | «Ты видишь»             |
| Будущее время   | «Я сделал» | «Я сделаю»              |
| Безличное       | «Я решил»  | «Мне пришлось решить»   |
| Мн. число       | «Ты был»   | «Вы были» (если группа) |

### 2. Обращение: ты / Вы

В английском всё `you`. В русском выбирай осознанно:

| Ситуация                          | Обращение                 |
| --------------------------------- | ------------------------- |
| Любовники, близкие, друзья        | ты                        |
| Незнакомцы, официоз               | Вы                        |
| Начальник / подчинённый           | Вы (или ты, если близкие) |
| Переход с Вы на ты во время секса | маркер сближения          |

**Переход с «Вы» на «ты» в интимной сцене — сюжетно обоснован.** Если в разгар секса вдруг возвращается «Вы» — это ошибка.

### 3. Адаптация междометий и стонов (КРИТИЧНО)

Английские звуки НЕ копировать буквально. Адаптировать под русскую фонетику.

#### Принципы адаптации стонов в диалоге

Стон может быть отдельным SFX (см. раздел SFX выше) или частью диалоговой реплики. Если это часть реплики — он подчинён тем же правилам, что и обычный текст:

1. **Длина и повтор букв** передаёт интенсивность. `Fhaaa..!` → `Ахха..!`, `Mnhhh!!` → `М-м-м!!`
2. **Точки и тильда** — паузы, сбивчивое дыхание. Сохранять: `Mnhhh... yess...` → `М-м-м... да...`
3. **~** в конце `Yesss...!~` — игривый/восходящий тон. Можно опустить или передать интонационно.
4. **Звуки с закрытым ртом** (`mngh`, `fhh`, `ngh`, `guhh`, `mnh`) — имитируют мычание/сдавленный звук. В русском: `м-м-м`, `ух`, `н-н-н`, `м-н-н`.
5. **Не переводить буквально** английские стоны как русские слова. `Fhaaa!!` ≠ «Фхаа!!» как транслит, а как естественный русский стон.
6. **ЗАПРЕЩЕННЫЕ КЛАСТЕРЫ: «Фх» и «Нх».** В русской фонетике нет сочетания «фх» или «нх» в начале слога. Английское `Fhaaa` — это придыхательный выдох с губно-зубного «f» (как начало `fuck`, переходящее в стон), а `Nhaah` — носовой «n» + выдох. Передача через «фх»/«нх» — грубая ошибка. Правильные аналоги:
   - `Fhaaa / Fhhaa` → `Ахха!` / `А-а-ах!` / `Уф-ф!`
   - `Nhaah / Nhaa` → `На-а!` / `А-а!` / `Н-на!`

7. **«Х» в стонах: когда можно, когда нет.** Русское «х» — велярный спирант (задняя стенка гортани). Английское `h` — просто выдох. Поэтому:
   - **Можно** (устоявшиеся русские междометия): `Ах!`, `Ох!`, `Ух!`, `Ха!`, `Хм`, `Кх!` (хрип/спазм)
   - **Нельзя** (псевдо-фонетика, нечитаемо): `М-мх`, `Ых`, `М-мха`. Везде заменять на простое гудение без «х»: `М-м-м`, `Н-н-н`, `На-а`

8. **«Fh-слова»: `Fhuuuuck` ≠ «Фу-у-ук».** Если английское слово начинается с `Fh` + слово (`Fhuuuuck`, `Fhhuck`, `Fhhucckkk`) — это не звук, а придыхательное произнесение слова (обычно `fuck`). **Не цепляй «Ф» впереди.** Просто переведи слово как есть: `Бля-а-а` / `Бляяя` / `Чёррт`. Добавление `Ф-ф-ф` или `Фх` создаёт нечитаемую конструкцию. Пример:
   - `Fhuuuuck` → `Бля-а-а` (а не `Фу-у-ук` или `Фху-у-ук`)
   - `Fhhucckkk` → `Бля-а-а` (а не `Ф-ф-ф... хуяккк`)
   - `Fhhuckhh` → `Бля-а-а` (а не `Ф-ф-ф... бля`)

   Исключение: если `Fhhh` — это отдельный короткий выдох (не часть слова), можно `Ффф`.

9. **ЗАПРЕЩЁННЫЕ КЛАСТЕРЫ: «гХ», «нгХ», «мгХ».** Сочетания `гх`, `нгх`, `мгх` в русском языке нечитаемы — это горловой спазм, которого нет в английских стонах. Английское `ngh` / `mngh` / `gh` — это носовой + выдох или гортанная смычка, а не «г» + «х». **Особое внимание при ревью:** проверять каждое вхождение `гх`, `нгх`, `мгх` в переводе. Правильные аналоги:
   - `nghhh` / `mnghhh` → `Н-н-н` / `М-м-м` (чистое гудение, без «гх»)
   - `mnhhh` → `М-н-н` (губно-носовой, без «гх»)
   - `ghuh` / `guhh` → `Ух!` / `Уух!` (короткий толчок)
   - `fghh` → `Ф-ф` (выдох, не «фгх»)
   - `ngh` внутри реплики — то же правило: `нгхх` → `н-н-н`
