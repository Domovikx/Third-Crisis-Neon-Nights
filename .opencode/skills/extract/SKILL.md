## Описание

Продвинутый поиск строк для перевода в Unity-файлах игры.
В отличие от `find-strings`, этот скил:

- Фильтрует шум (шейдеры, пути, константы, base64)
- Разделяет диалоговый текст и UI-строки по группам
- Сохраняет в отдельные файлы

## Когда использовать

- Для первичного сбора всех строк игры
- После обновления игры — найти новый непереведённый текст
- Для составления полного словаря (корпуса) игры

## Скрипт

```bash
node .opencode/skills/extract/extract.mjs
node .opencode/skills/extract/extract.mjs --min 30
node .opencode/skills/extract/extract.mjs --dir "Third Crisis Neon Nights_Data"
```

## Где хранится текст в игре

### 1. Level файлы (level0-level15) — ОСНОВНОЙ ИСТОЧНИК

Бинарные Unity-сцены. Содержат:

- **TextMeshPro / UGUI Text** — диалоги, UI-надписи
- **PlayMaker FSM** — строковые переменные в визуальных скриптах
- **MonoBehaviour** — кастомные скрипты с текстом (NToolkit)

Формат: бинарный Unity serialized. Парсится собственным скриптом extract.mjs
(бинарный поиск ASCII + чтение length-prefixed строк).

### 2. Shared assets (sharedassets0-15.assets)

Содержат спрайты, текстуры, шрифты, префабы.
Текст может быть внутри префабов как default-значения компонентов.

### 3. StreamingAssets (Addressables бандлы)

97 бандлов с артом, анимациями, аудио.
Текст внутри MonoBehaviour-конфигов (пинапы, позы).

### 4. Managed DLL (Assembly-CSharp.dll)

Код игры. Содержит строки UI, названия квестов, описания,
но большинство текста — в сценах.

## Алгоритм извлечения

```
Чтение бинарного файла
  ↓
Поиск ASCII-строк (от MIN_LEN символов)
  ↓
Фильтр #1: букв > 40%, заглавных < 70%
  ↓
Фильтр #2: отсев шума (шейдеры, пути, константы)
  ↓
Группировка: диалоговые (длинные, lower case) / UI (короткие, upper case)
  ↓
Сохранение в extracted-*.txt
```

## Результат

Файлы в `.opencode/skills/extract/`:

| Файл                     | Описание                                 |
| ------------------------ | ---------------------------------------- |
| `extracted-all.txt`      | Все найденные строки                     |
| `extracted-dialogue.txt` | Диалоговые строки (длинные, с пробелами) |
| `extracted-ui.txt`       | UI-элементы (кнопки, заголовки)          |

## Ограничения текущего подхода

- Структура MonoBehaviour не известна статически — строки извлекаются бинарным поиском
- PlayMaker FSM строки — только через бинарный поиск
- Для точного извлечения нужен рантайм-плагин (BepInEx), работающий в игре
