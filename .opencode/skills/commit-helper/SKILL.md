# commit-helper — Оформление коммитов

Анализирует изменения в репозитории и генерирует commit message в формате Conventional Commits на русском языке.

## Workflow

1. Запросить `git status` и `git diff --cached --stat` — посмотреть что в индексе
2. Запросить `git diff --cached` — полный diff закешированных изменений
3. Если индекс пуст, запросить `git diff HEAD --stat` — непроиндексированные изменения
4. Определить:
   - **type** — тип изменений (см. таблицу)
   - **scope** — модуль проекта: extractor, translator, proxy, fonts, runtime, build, deploy, yaml, docs, deps
   - **description** — заголовок на русском, повелительное наклонение, макс. 72 символа
   - **body** — подробности: ЧТО и ПОЧЕМУ, не КАК (опционально)
   - **footer** — Breaking changes, Closes #issue (опционально)
5. Вывести готовый commit message пользователю для ручного использования
6. **ВАЖНО: весь commit message — только на русском языке. Заголовок и body. Английский недопустим.**

## Типы изменений

| Type     | Когда использовать                                    |
| -------- | ----------------------------------------------------- |
| feat     | Новая функциональность, новый файл, новая возможность |
| fix      | Исправление бага, ошибки, краша                       |
| refactor | Переписывание кода без изменения поведения            |
| docs     | Документация, README, AGENTS.md, комментарии          |
| style    | Форматирование, отступы, запятые (не CSS)             |
| test     | Новые тесты или правка существующих                   |
| chore    | Обслуживание: deps, конфиги, gitignore                |
| perf     | Оптимизация производительности                        |
| ci       | Изменения в CI/CD                                     |
| build    | Система сборки, скрипты, Python/C# компиляция         |

## Правила хорошего коммита

1. **Весь коммит на русском** — заголовок, body, footer. Никакого английского.
2. **Заголовок** — повелительное наклонение, с маленькой буквы, без точки в конце
   - ✅ `feat(extractor): добавить парсинг rich_text из JSON`
   - ❌ `fix(extractor): resolve color tags` (английский)
   - ❌ `feat(extractor): Добавил парсинг rich_text.`
3. **Длина заголовка** — до 72 символов (лучше до 50)
4. **Body** — отвечает на вопросы ЧТО и ПОЧЕМУ, не КАК, только на русском
5. **Body** — пустая строка между заголовком и телом
6. **Breaking change** — `!` после type/scope: `feat(api)!: удалить старый endpoint`
6. **Scope** — модуль проекта (см. AGENTS.md):
   - `extractor` — скрипты извлечения текста
   - `translator` — рантайм-переводчик (C#)
   - `proxy` — dwmapi.dll native proxy
   - `fonts` — шрифты, кириллица
   - `runtime` — DLL рантайма
   - `build` — скрипты сборки
   - `deploy` — деплой переводов
   - `yaml` — файлы переводов
   - `docs` — документация
   - `deps` — зависимости

## Примеры

```
feat(extractor): добавить поддержку .bundle файлов

Переписан загрузчик на UnityPy v2, добавлен парсинг raw_strings
для PlayMaker FSM в .bundle ассетах.
```

```
fix(translator): исправить краш при пустом translation

TranslationLoader падал с NullReferenceException если в YAML
встречалась пустая строка перевода. Добавлена проверка на null.
```

```
refactor(build): объединить build.py и build_proxy.py

Вынесена общая логика поиска компиляторов в shared.py.
Упрощена обработка путей через pathlib вместо os.path.
```

```
chore(deps): обновить UnityPy до 2.15.0
```

```
docs(readme): обновить инструкцию по установке шрифтов

Добавлен раздел про кириллический шрифт, исправлены устаревшие пути.
```

```
feat(translator)!: перейти на новую схему словаря

BREAKING CHANGE: Ключи словаря теперь используют path_id вместо
raw_text. Старые YAML-файлы несовместимы.
```
